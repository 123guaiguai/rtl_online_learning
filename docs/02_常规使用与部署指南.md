# 常规使用与部署指南

## 一、环境准备

### 1.1 系统依赖

| 依赖 | 版本要求 | 安装方式 |
|------|----------|----------|
| **Icarus Verilog** | v10.3+ | `sudo apt-get install -y iverilog` |
| **Python** | 3.10 | 通过 conda 管理 |
| **Conda** | 任意版本 | [Miniconda 官网](https://docs.conda.io/en/latest/miniconda.html) |

### 1.2 创建 Python 环境

```bash
# 创建 conda 环境
conda create -n autochip python=3.10 -y
conda activate autochip

# 安装 Python 依赖（国内镜像加速）
cd /home/gq/Autochip_workspace/Experiments
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

`requirements.txt` 中的主要依赖：
- `openai`：调用 OpenAI 兼容接口（Siliconflow、GitHub Models 等）
- `anthropic`：调用 Claude 系列模型
- `google-generativeai`：调用 Gemini 系列模型
- `regex`：正则表达式扩展库

## 二、API 密钥配置

本项目使用外部配置文件管理 API 密钥，**密钥不会随代码提交到 Git 仓库**。

### 2.1 创建密钥配置文件

```bash
cd /home/gq/Autochip_workspace/Experiments

# 复制示例文件
cp api_keys.example.py api_keys.py

# 编辑文件，填入真实密钥
nano api_keys.py
```

### 2.2 密钥文件内容

`api_keys.py` 文件格式如下：

```python
# Siliconflow (硅基流动) API 配置
SILICONFLOW_CONFIG = {
    'api_key': 'sk-your-real-siliconflow-key',   # 替换为真实密钥
    'base_url': 'https://api.siliconflow.cn/v1'
}

# GitHub Models API 配置（使用 GitHub Token）
GITHUB_CONFIG = {
    'api_key': 'ghp-your-real-github-token',      # 替换为真实 Token
    'base_url': 'https://models.inference.ai.azure.com'
}
```

### 2.3 安全注意事项

> ⚠️ **`api_keys.py` 已被加入 `.gitignore`，不会提交 Git，请不要手动将其 add 到仓库。**

- 不要在代码或日志中打印完整密钥
- 定期轮换密钥（建议每 3-6 个月）
- 如果密钥泄露，立即在 API 平台控制台撤销并重新生成

### 2.4 验证密钥配置

```bash
conda run -n autochip python -c "
import sys; sys.path.insert(0, 'core')
from languagemodels import ChatGPT
print('密钥加载成功')
"
```

## 三、单题运行（快速验证）

### 3.1 使用快速启动脚本（推荐新手）

```bash
cd /home/gq/Autochip_workspace/Experiments

# 语法：./run_demo.sh <测试用例名> <模型提供商>
./run_demo.sh rule90 siliconflow
```

该脚本会：
1. 检查 `autochip` conda 环境是否存在
2. 自动激活 conda 环境
3. 清理代理设置
4. 运行 `generate_verilog.py` 并在 `outputs/` 目录下保存结果

### 3.2 手动运行单题

```bash
# 1. 激活环境
conda activate autochip

# 2. 进入脚本目录（必须在此目录内运行，脚本使用相对路径）
cd /home/gq/Autochip_workspace/Experiments/core

# 3. 清理代理（避免 SOCKS 代理干扰 API 请求）
unset http_proxy https_proxy all_proxy

# 4. 运行指定配置文件
python generate_verilog.py -c configs/config_siliconflow.json
```

### 3.3 JSON 配置文件说明

`configs/` 目录下存放各题目的配置文件，格式如下：

```json
{
    "general": {
        "prompt": "../VerilogEval/Prob001_zero_prompt.txt",
        "name": "TopModule",
        "testbench": "../VerilogEval/Prob001_zero_test.sv",
        "model_family": "Siliconflow",
        "model_id": "Qwen/Qwen2.5-Coder-32B-Instruct",
        "num_candidates": 1,
        "iterations": 3,
        "outdir": "outputs/test_Prob001",
        "log": "log.txt",
        "mixed-models": false
    },
    "mixed-models": {}
}
```

| 字段 | 说明 |
|------|------|
| `prompt` | 题目描述文件路径（作为 LLM 提示词） |
| `name` | 模块名称（生成的 sv 文件名前缀） |
| `testbench` | 仿真测试台文件路径 |
| `model_family` | LLM 提供商（`Siliconflow` / `ChatGPT` / `Claude` / `Gemini`） |
| `model_id` | 具体模型 ID |
| `num_candidates` | 每次迭代生成的候选数量（越多成功率越高，API 消耗越大） |
| `iterations` | 最大迭代次数（实际执行 iterations+1 次，从 iter0 开始） |
| `outdir` | 输出目录路径 |
| `mixed-models` | 是否启用混合模型策略 |

### 3.4 查看运行结果

```bash
# 查看主日志（含最终 Rank）
cat outputs/test_Prob001/log.txt

# 查看生成的 Verilog 代码
cat outputs/test_Prob001/iter0/response0/TopModule.sv

# 搜索最终 Rank 值
grep "Rank of best response" outputs/test_Prob001/log.txt
```

**日志关键字段解读：**

```
Time to Generate: 5.26           ← 生成总用时（秒）
Best ranked response at iteration 0 with response number 0
Rank of best response: 1.0       ← 最终 Rank（1.0 = 通过）
```

## 五、常见问题与解决

### 问题 1：代理导致 API 请求失败

**现象：** 请求超时或返回 SOCKS 代理错误

**解决：**
```bash
unset http_proxy https_proxy all_proxy HTTP_PROXY HTTPS_PROXY ALL_PROXY
```
代码内部也会自动清理代理，但建议在 shell 层面也清理。

### 问题 2：密钥配置文件找不到

**现象：**
```
❌ 错误: 未找到API密钥配置文件 api_keys.py
```

**解决：**
```bash
cd /home/gq/Autochip_workspace/Experiments
cp api_keys.example.py api_keys.py
# 然后编辑文件填入真实密钥
```

### 问题 3：iverilog 未安装

**现象：** `iverilog: command not found`

**解决：**
```bash
sudo apt-get update && sudo apt-get install -y iverilog
iverilog -V  # 验证安装
```

### 问题 4：conda 环境激活失败

**解决：**
```bash
# 初始化 conda（如果 conda activate 报错）
conda init bash
source ~/.bashrc

# 重新创建环境
conda create -n autochip python=3.10 -y
conda activate autochip
pip install -r requirements.txt
```

### 问题 5：编译始终失败（Rank = -1）

**可能原因与建议：**
- 模型能力不足 → 尝试切换到更强的模型
- 迭代次数不够 → 增加 `iterations` 值
- 题目描述不清晰 → 检查 prompt 文件内容
- 混合模型配置错误 → 检查 `mixed-models` 配置

## 六、项目文件重要说明

### 运行目录的选择

**必须从 `core/` 目录运行脚本**，因为配置文件中使用了如 `../VerilogEval/` 这样的相对路径。

```bash
# ✅ 正确
cd /home/gq/Autochip_workspace/Experiments/core
python generate_verilog.py -c configs/xxx.json

# ❌ 错误（相对路径无法正确解析）
cd /home/gq/Autochip_workspace
python Experiments/core/generate_verilog.py -c ...
```

### 输出文件清理

如果 `outputs/` 目录积累了太多旧的运行结果，可以手动清理：

```bash
# 查看输出目录大小
du -sh Experiments/core/outputs/

# 清空旧结果（谨慎操作！）
rm -rf Experiments/core/outputs/batch_tests/old_experiment_name
```

# RTL-Tutor 平台手动验收与测试用例手册

本手册旨在指导您手动验证 RTL-Tutor 平台的各项功能。请确保您已经完成了本地 SSH 端口转发，并使用 Chrome 浏览器打开了以下前端地址：
👉 **`http://localhost:5173`**

---

## 准备工作：连接与端口转发

如果您是在本地 Windows 机器上操作，请通过以下 SSH 命令连接服务器（以开启双向通道）：
```bash
ssh -L 5173:localhost:5173 -L 8000:localhost:8000 your_username@server_ip
```
> [!NOTE]
> 此时，您的本地浏览器输入 `http://localhost:5173` 会代理到服务器的前端 Vite，而前端发送给 `http://localhost:8000` 的 API 请求会代理到服务器的 FastAPI 后端。

## 测试用例 1：题目检索与切换验证

### 🎯 测试目的
验证题目加载、模糊搜索以及模板代码的自动生成。

### 📋 步骤与预期结果
1. **加载页面**：
   * 打开 `http://localhost:5173`。
   * **预期结果**：左侧出现题目列表，默认选中第一题 `Prob001 zero`。左下方显示“接口与设计说明”，中间 Monaco 编辑器自动载入对应的 SystemVerilog 端口定义模板，右侧 AI 助教弹出欢迎语。
2. **关键词过滤**：
   * 在左上角输入框中输入 `vector`。
   * **预期结果**：列表自动过滤，仅保留名称中含有 `vector` 的题目（如 `Prob004 vector2`，`Prob006 vectorr` 等）。
3. **切换题目**：
   * 搜索框清空，点击题目列表中的 **`Prob003 step_one`**。
   * **预期结果**：Monaco 编辑器内的代码模板和左下角的“接口与设计说明”自动更新为 `step_one` 的接口规范；右侧 AI 会话自动清除旧历史，重新弹出该题目的专属向导。

## 测试用例 2：Monaco 实时语法校验（Linter）

### 🎯 测试目的
验证编辑器在非编译状态下，打字时的实时语法纠错与清晰易读的语法错误提示。

### 📋 步骤与预期结果
1. 切换回第一题 **`Prob001 zero`**。
2. 在中间编辑器中输入以下代码（**故意删掉 `assign` 语句行末的分号**）：
   ```verilog
   module TopModule (
     output zero
   );
     assign zero = 1'b0
   endmodule
   ```
3. 静置编辑器 **1-2 秒**（触发防抖校验机制）。
4. **预期结果**：
   * 编辑器第 5 行的 `endmodule` 下方自动绘制出红色波浪线。
   * 将鼠标悬停在红色波浪线上，会弹出浮窗提示：`语法错误 (syntax error)：请检查该行或前一行是否缺少分号 ';'，或者括号/分号/关键字是否配对。`。

## 测试用例 3：Docker 沙箱编译与仿真（成功通过案例）

### 🎯 测试目的
验证输入正确逻辑时，Docker 隔离环境编译、测试矢量比对、仿真通过率指示和 VCD 时序波形渲染的联合连通性。

### 📋 步骤与预期结果
1. 在 **`Prob001 zero`** 题目下，将编辑器中的代码修复为**完全正确**的逻辑：
   ```verilog
   module TopModule (
     output zero
   );
     assign zero = 1'b0;
   endmodule
   ```
2. 点击中间编辑器上方的 **“运行测试”** 按钮。
3. **预期结果**：
   * 按钮变为加载态，底部控制台面板自动切换到 **“输出日志 (Console Logs)”**，打印沙箱启动和运行日志。
   * 仿真结束后（约 1-2 秒），控制台标签自动切到 **“仿真指标 (Simulation Metrics)”**。
   * 编译状态显示 `SUCCESS`，仿真通过率显示 `100.0%`，进度条填充为紫色渐变，参数显示 `共 20 周期 / 错 0 周期`。
4. 点击底部控制台的 **“时序波形 (Waveforms)”** 标签。
5. **预期结果**：
   * 渲染出逻辑分析仪视图。左侧显示信号 `clk`、`tb_mismatch`、`zero_ref` 和 `zero_dut`。
   * 右侧时间轴展示时钟信号 `clk` 呈青色周期方波变化；错误信号 `tb_mismatch`、参考信号与设计输出信号均为 `0`，没有错位。

## 测试用例 4：仿真对齐失败案例（错误逻辑）

### 🎯 测试目的
验证逻辑有误时，测试比对失败、通过率按比扣减和波形图红色报错提示。

### 📋 步骤与预期结果
1. 在 **`Prob001 zero`** 下，将逻辑语句修改为**错误电平**：
   ```verilog
   module TopModule (
     output zero
   );
     assign zero = 1'b1; // 题目要求始终输出 0，此处故意写为 1
   endmodule
   ```
2. 点击 **“运行测试”** 按钮。
3. **预期结果**：
   * 编译状态显示 `SUCCESS`。
   * 仿真通过率降为 `0.0%`。
   * 底部指标显示：`共 20 周期 / 错 20 周期`。
4. 点击 **“时序波形 (Waveforms)”** 标签页。
5. **预期结果**：
   * 信号 `tb_mismatch`（不匹配信号）电平拉高变为 `1`，且折线在波形中渲染为**亮红色**。
   * 信号 `zero_ref` 为低电平 `0`，而学生设计的输出 `zero_dut` 为高电平 `1`，波形清晰比对出错误并展示出来。

## 测试用例 5：AI 助教流式引导式提问与答疑

### 🎯 测试目的
验证 DeepSeek-V3 在硬件启发式教学约束下的错误诊断、智能识别自定义提问并直接答疑，以及流式返回效果。

### 📋 步骤与预期结果
1. 保持在刚刚产生 `20个周期不匹配` 的状态下，点击编辑器右上方的 **“AI 助教诊断”** 按钮。
2. **预期结果**：
   * 右侧聊天框自动滑入一条学生消息：“助教，我遇到了仿真报错，请帮我分析...”。
   * 接着，AI 助教开始以 **打字流式效果（Streaming）** 连续输出中文字符。
   * AI 能够精准识别当前的题目是 `Prob001-zero` 并且指出仿真出现了 20 个 mismatch，以启发式逻辑（如：“你的输出 zero 一直保持高电平，但题目要求它输出低电平...”）引导修改，而不是直接给出整段代码。
3. 点击右下方输入框上方的快捷提问键 **“💡 原理说明”**。
4. **预期结果**：
   * 聊天框自动发送“解释这道题目所要求的硬件电路原理...”。
   * AI 再次进行流式回复，详细阐述该电路的逻辑学特性。
5. 在右下角文本框中手动输入：`Verilog 中 = 和 <= 有什么区别？` 并回车。
6. **预期结果**：
   * 消息发出，AI 助教直接进入**答疑模式**，聚焦并专业地回答 `=`（阻塞）与 `<=`（非阻塞）的语法及物理硬件区别，排版精美。

## 测试用例 6：多轴拖拽布局与个性化设置 (Theme/Font/Resize)

### 🎯 测试目的
验证三栏宽度手动调整、控制台高度调节、提问区拖拽伸缩、字号大小滑动调节及明暗主题的无缝切换。

### 📋 步骤与预期结果
1. **左右三栏拖拽**：
   * 将鼠标悬停在左侧面板与中间编辑区之间的细缝（或中间编辑区与右侧 AI 面板之间的细缝）。
   * 按住鼠标左键并左右拖动。
   * **预期结果**：左侧/右侧面板宽度随鼠标移动实时扩展或收缩（限制在安全的 min/max 范围内，防止过度挤压），中间代码编辑区自适应伸缩。
2. **控制台高度拖拽**：
   * 将鼠标悬停在中间编辑器与底部控制台之间的分界把手 `.console-resizer-handle` 上。
   * 按住鼠标左键并上下拖动。
   * **预期结果**：控制台高度实时改变（限制在 120px - 600px），编辑器高度自适应更新。
3. **提问区整体拖拽缩放**：
   * 将鼠标悬停在右侧 AI 对话框底部的输入区上边界 `.chat-input-resizer-handle` 上。
   * 按住鼠标左键并上下拖动。
   * **预期结果**：提问区总高度实时更新（限制在 120px - 400px），同时输入文本框会自动按比例纵向拉伸大小，方便编写长句。
4. **手动字号调节**：
   * 拖动顶部 Header 中的 **“字号”** 滑块。
   * **预期结果**：编辑器 Verilog 代码字体、描述文本、AI 对话及控制台日志字体同步平滑变大或变小（滑块标签显示实时像素字号）。
5. **明暗主题切换**：
   * 点击 Header 中字号滑块右侧的 **“太阳/月亮”** 图标按钮。
   * **预期结果**：
     * 整个应用切换为高对比度、符合无障碍阅读的 **浅色模式（Light Mode）**。
     * Monaco 编辑器自动载入标准的 VS Light 主题。
     * 所有面板背景、文字颜色同步切换，保障白天明亮环境下的阅读对比度。
